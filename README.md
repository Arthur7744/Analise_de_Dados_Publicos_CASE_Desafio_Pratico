# Analise_de_Dados_Publicos_CASE_Desafio_Pratico
Desafio de análise de dados públicos e desenvolvimento de dashboards e insights sobre o tema.
Todas as etapas percorridas estão descritas no arquivo PDF "Relatório de Case - Processo Seletivo para Dados.pdf".
Além do arquivo PDF, esse repositório conta com: 
- as duas bases de dados utilizadas para a análise dos dados sendo elas os arquivos "estimativaAnoQtdeHospitais(in).csv" e "estimativaMunicipiosMG(in).csv";
- os códigos feitos na IDE Jupyter Notebook que foram necessários durante o projeto("CasePTCI.ipynb");
- a base de dados já integrada após o tratamento dos dados no formato xlsx("base_final.xlsx");
- e o dashboard feito após a base de dados tratada("deshboardCase.pbix").

## DESCRIÇÃO DO DESAFIO
1. OBJETIVO
Avaliar a capacidade do(a) candidato(a) em estruturar, integrar, analisar e comunicar dados
voltados à gestão pública, com foco em geração de valor para tomada de decisão.
2. CONTEXTO
O profissional deverá atuar na estruturação de uma base de dados integrada que apoie decisões
estratégicas no contexto de cidades inteligentes e gestão pública municipal.
3. BASES DE DADOS
O candidato deverá utilizar, no mínimo, duas das seguintes fontes públicas de dados: IBGE
(população), DATASUS (saúde) e INEP (educação). Recomenda-se a utilização de dados em
formato CSV.
4. ETAPAS DO DESAFIO
4.1 Ingestão e Tratamento de Dados
Integração de diferentes fontes de dados, tratamento de inconsistências, padronização de chaves
(ex.: município, ano) e documentação das decisões adotadas.
4.2 Modelagem de Dados
Definição de um modelo de dados que permita análises temporais e comparações entre
municípios. Recomenda-se a apresentação de um diagrama (ex.: modelo estrela).
4.3 Pipeline de Dados
Implementação de um fluxo de processamento de dados (ex.: Python, SQL ou ferramentas
equivalentes), garantindo reprodutibilidade e organização.
4.4 Análise de Dados
Apresentação de 3 a 5 insights relevantes para apoio à gestão pública. Espera-se capacidade
analítica além da simples descrição de dados.
4.5 Visualização de Dados
Construção de dashboards ou visualizações que facilitem a interpretação e a tomada de decisão.
4.6 Documentação
Elaboração de documentação contendo decisões técnicas, limitações do trabalho e sugestões de
evolução da solução proposta.
5. CRITÉRIOS DE AVALIAÇÃO
1 Qualidade da modelagem de dados;
2 Organização e clareza do pipeline desenvolvido;
3 Consistência no tratamento e integração dos dados;
4 Capacidade analítica e geração de insights;
5 Clareza na comunicação e documentação.
6. ENTREGA
O candidato deverá entregar repositório (ou arquivo compactado) contendo o código-fonte, dados
(ou instruções para obtenção), visualizações e documentação do projeto.
